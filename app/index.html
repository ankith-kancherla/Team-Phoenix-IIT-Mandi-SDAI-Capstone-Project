How APIPhoenix worksA clear path from raw route definitions to output a user can act on.

STEP 01
📥
Ingest

Upload an endpoints.json, add routes by hand, paste a cURL command, or start from 49 bundled sample endpoints across 5 services.

STEP 02
🧠
Understand

Claude AI reads each contract and writes readable documentation. A deterministic local engine guarantees output even fully offline.

STEP 03
▷
Try

The Playground builds copy-paste samples in cURL, JavaScript and Python, and simulates realistic success and failure responses.

STEP 04
◉
Test

The Test Lab generates a contract-grounded suite per endpoint and runs it with a live pass/fail report.

STEP 05
⊘
Act

Error Doctor turns any raw error into a diagnosis and concrete next steps — so the user always knows what to do next.

STEP 06
◍
Measure

The Insights dashboard tracks docs and test coverage live, per service, and exports the whole surface as OpenAPI 3.0.

AI settings

When this file is opened locally, browsers block keyless AI calls (CORS) — the portal then runs on the built-in Phoenix engine, which covers every feature offline. To enable live Claude generation, paste an Anthropic API key.
